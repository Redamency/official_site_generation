# Distillation

## 1. 什么是蒸馏

蒸馏（distillation, 或 knowledge distillation）指的是：用一个更强、更大、成本更高的老师模型（teacher），去指导一个更小、更快、部署更便宜的学生模型（student）训练。

它的核心不是单纯“抄答案”，而是让学生去逼近老师的行为分布、表示空间或推理偏好。相比只用人工标注的硬标签训练，蒸馏通常能提供更丰富的监督信号。

常见目标包括：

- 模型压缩：把大模型能力迁移到小模型
- 加速推理：降低延迟、显存和部署成本
- 提升泛化：学生有时会比直接监督训练更稳
- 迁移能力：把一个领域、任务或策略迁移到另一模型
- 稳定训练：在强化学习、多任务学习中作为额外监督

## 2. 蒸馏为什么有效

老师模型输出的不只是“正确类别”，还包含类间相似性、置信度结构和决策边界信息。

例如在分类任务里：

- 硬标签只告诉学生“这是猫，不是狗”
- 老师的软标签可能告诉学生“这是猫的概率 0.82，狐狸 0.10，狗 0.06，其他很低”

这类软目标通常包含了更细的结构信息，因此学生更容易学到更平滑、更合理的决策边界。

## 3. 基本形式

最经典的蒸馏来自 Hinton 等人的 soft target distillation。设老师 logits 为 `z_t`，学生 logits 为 `z_s`，温度为 `T`，则常见目标是：

```text
L = alpha * L_hard + beta * T^2 * KL(softmax(z_t / T) || softmax(z_s / T))
```

其中：

- `L_hard` 是学生对真实标签的监督损失，通常是交叉熵
- 第二项是蒸馏损失，要求学生匹配老师的输出分布
- `T > 1` 时，概率分布更平滑，更能暴露“次优类别”的相对关系

实践里常见做法是同时保留：

- 对真实数据标签的监督
- 对老师输出分布的模仿

这样学生既不完全依赖老师，也不丢掉原始任务目标。

## 4. 常见蒸馏类型

### 4.1 Logit 蒸馏

这是最常见、最基础的一类蒸馏。学生直接拟合老师在输出层的 logits 或 softmax 分布。

典型方法：

- KL 散度蒸馏：匹配老师和学生的输出分布
- Logit 回归：直接回归 logits
- 温度蒸馏：通过温度 `T` 拉平分布，让暗知识更明显

优点：

- 实现简单
- 适用范围广
- 对分类、语言模型 next-token 预测都很自然

局限：

- 只约束最终输出，不直接约束中间表示
- 如果老师本身校准较差，学生可能继承错误偏好
- 当老师和学生架构差异很大时，单靠输出蒸馏可能不够

在大语言模型里，logit 蒸馏通常对应：

- 让学生在每个 token 位置拟合老师的 next-token 分布
- 而不是只学习老师最终生成的文本字符串

这通常比单纯 SFT 老师生成的数据更细，因为它保留了完整词表上的概率结构。

### 4.2 Feature / Representation 蒸馏

除了输出层，还可以让学生对齐老师的中间层表示，例如 hidden states、attention map、特征图或某一层投影空间。

常见做法：

- L2 对齐中间层表示
- 匹配注意力矩阵
- 匹配 embedding 空间
- 使用 projector 把老师和学生投到同一维度再对齐

适用情况：

- CNN、ViT、Transformer 等深层网络
- 老师中间表示质量高，希望学生继承表征能力

优点：

- 约束更强
- 往往比只蒸馏输出更稳定

难点：

- 需要处理层数不同、宽度不同的问题
- 中间层并不一定一一对应

### 4.3 Relation Distillation

有些方法不要求学生逐点复制老师特征，而是要求保留样本之间、类别之间或 token 之间的关系结构。

例如：

- 样本间距离矩阵对齐
- 类别相似度结构对齐
- token-token 关系蒸馏

这种方法本质上更关注“结构”而不是“数值逐元素相等”。

### 4.4 Sequence-Level 蒸馏

在机器翻译、摘要、对话、代码生成等生成任务中，常用 sequence-level distillation：

- 先让老师模型生成整句或整段结果
- 再把这些老师生成的序列当作学生的训练目标

这类方法很实用，因为它把复杂分布压缩成了一个更易学习的目标分布。学生不用面对原始数据中多种可能的正确答案，而是学一个更“规范”的老师输出。

优点：

- 工程简单
- 对生成任务非常有效

缺点：

- 会丢掉老师完整的 token-level 分布信息
- 容易把老师的风格和偏差一起固化

## 5. 数据蒸馏

“数据蒸馏”这个词在文献和工业实践里有两种常见含义，需要区分。

### 5.1 用老师生成训练数据

这是工程上最常见的含义：用强老师模型产生高质量训练样本，再用这些样本训练学生。

例如：

- 老师为原始 prompt 生成更好的回答
- 老师补充 chain-of-thought、理由、解释、标签
- 老师做数据清洗、去噪、重写、难例扩充
- 老师对无标签样本打伪标签

这种方法本质上是“teacher-generated data”或“synthetic data for distillation”。

优点：

- 非常实用
- 容易扩展到海量语料
- 对 LLM 蒸馏尤其常见

风险：

- 数据分布会被老师风格主导
- 老师幻觉会被批量写进数据
- 如果只学老师文本而不学分布，学生可能更像在做 imitation，而不是严格意义上的 logit KD

### 5.2 Dataset Distillation / Dataset Condensation

另一种更严格的“数据蒸馏”指的是：把一个大训练集压缩成一个很小的合成数据集，使得模型只在这小批数据上训练，也能接近原始大数据集的效果。

它关注的不是“老师到学生”，而是“数据到小数据”的压缩。常见方法包括：

- 梯度匹配
- 训练轨迹匹配
- 分布匹配
- 生成式样本优化

目标是得到少量但高信息密度的样本。

适用场景：

- 数据选择与压缩
- 快速实验
- 边缘设备训练
- 隐私敏感场景下的数据摘要

需要注意：

- dataset distillation 和 knowledge distillation 有交集，但不是一回事
- 前者压缩的是训练数据，后者压缩的是模型知识

## 6. On-Policy Distillation

on-policy distillation 主要出现在强化学习和交互式决策问题中，也逐渐被借用到在线生成式训练场景。

它的核心思想是：

- 训练数据不是固定离线采样出来的
- 而是由当前策略或当前学生在线产生
- 然后再让老师在这些“当前分布”的状态、轨迹或 prompt 上给出指导

也就是说，蒸馏监督跟着当前策略访问到的数据分布一起变化。

### 6.1 与 off-policy distillation 的区别

off-policy distillation 通常是：

- 先收集一批固定数据
- 老师在这批数据上打标签
- 学生反复离线训练

on-policy distillation 则更像：

- 学生或当前策略先与环境交互
- 得到当前会访问到的状态或样本
- 老师再对这些状态给出动作分布、价值信息或偏好监督
- 学生继续更新

### 6.2 为什么需要 on-policy

在强化学习中，状态分布取决于策略本身。如果学生策略变了，它访问到的状态也会变。

因此如果蒸馏始终依赖旧数据，常见问题是：

- 分布偏移严重
- 学到的是过时状态下的老师行为
- 在真实运行分布上效果下降

on-policy distillation 的优点是更贴近学生当前会遇到的真实决策分布。

### 6.3 代价与难点

- 训练成本高，因为老师需要在线参与标注或指导
- 数据管线更复杂
- 在线采样噪声大，训练更不稳定
- 如果老师推理很贵，整体成本可能很高

### 6.4 在 LLM 语境下如何理解

在语言模型里，一个可理解的类比是：

- 不只对静态语料做蒸馏
- 而是让学生按当前策略生成回答或中间轨迹
- 再让老师对这些当前生成分布上的样本给反馈

这类做法在思想上更接近在线偏好优化、DAgger 式数据聚合或 rollout-based distillation，而不是传统静态 KD。

因此在写作时最好把 on-policy distillation 说成“一类在线、分布随当前策略变化的蒸馏范式”，而不是把它狭义限定为某一个固定算法。

## 7. Self-Distillation

self-distillation 指老师和学生并不是两个完全独立训练出来的不同模型，而是同一个模型体系内部进行知识迁移。

常见形式包括：

- 同架构前一代模型蒸馏后一代模型
- 同一个模型的深层指导浅层
- 同一个模型的历史 checkpoint 指导当前 checkpoint
- 多分支网络中强分支指导弱分支
- 同架构、同容量模型一代代递归蒸馏

### 7.1 为什么有效

这看起来有些反直觉：如果老师和学生容量差不多，为什么还会有收益？

常见解释包括：

- 蒸馏起到了正则化作用
- 训练过程中的不稳定噪声被平滑了
- 老师提供了比硬标签更丰富的软约束
- 多代训练能逐步改善决策边界

“Born-Again Networks” 是 self-distillation 的经典例子之一：学生和老师容量接近甚至相同，但学生仍可能超过老师。

### 7.2 常见变体

- Temporal self-distillation：历史模型指导当前模型
- Layer-wise self-distillation：深层指导浅层
- Online self-distillation：同一次训练中动态产生 teacher signal
- Multi-view / multi-branch self-distillation：不同视角相互蒸馏

### 7.3 注意事项

- self-distillation 不一定用于压缩模型
- 它很多时候更像提升精度和泛化的训练技巧
- 如果监督过强，也可能限制学生探索不同解

## 8. 蒸馏在大语言模型中的常见做法

LLM 蒸馏通常不是单一技术，而是几种手段组合：

- 用强老师生成 instruction-following 数据
- 用老师 logits 做 token-level 蒸馏
- 用偏好模型或 reward model 做对齐蒸馏
- 用 CoT、程序轨迹、工具调用轨迹做过程蒸馏
- 用在线 rollout 做 on-policy 或近似 on-policy 蒸馏

常见训练对象包括：

- base model 到小型 base model
- chat model 到更小的 chat model
- reasoning model 到轻量 reasoning model
- MoE / 大 dense 模型到小 dense 模型

实践中的一个关键判断是：你到底想蒸馏什么。

可能蒸馏的是：

- 最终答案
- token 概率分布
- 推理过程
- 工具使用习惯
- 偏好对齐后的行为风格

目标不同，训练设计也会明显不同。

## 9. 蒸馏的收益与风险

### 9.1 收益

- 更低推理成本
- 更快速度
- 更小显存占用
- 在小模型上获得超出纯监督训练的性能
- 把难以直接学到的行为模式迁移给学生

### 9.2 风险

- 学生继承老师错误
- 过度模仿导致创新能力下降
- 数据分布偏移时效果明显下降
- 蒸馏数据质量不稳时会放大噪声
- 老师过强、学生过弱时，优化可能困难

尤其在生成任务里，蒸馏常见的失败模式包括：

- 风格过拟合
- 回复模板化
- 长文本推理能力下降
- 只会模仿老师表面格式，学不到真正决策机制

## 10. 怎么选择蒸馏方案

可以按目标来选：

### 10.1 如果目标是模型压缩

优先考虑：

- logit 蒸馏
- sequence-level 蒸馏
- 少量中间层蒸馏

### 10.2 如果目标是提升生成质量

优先考虑：

- 老师生成高质量训练数据
- token-level 分布蒸馏
- 过程蒸馏

### 10.3 如果目标是强化学习或交互决策

优先考虑：

- policy distillation
- on-policy distillation
- value / action distribution distillation

### 10.4 如果目标是同模型提效或提精度

优先考虑：

- self-distillation
- temporal self-distillation
- online self-distillation

## 11. 几个容易混淆的概念

### 11.1 蒸馏 vs 剪枝

- 蒸馏是知识迁移
- 剪枝是删参数、删通道、删层

两者可以联合使用：先剪枝，再蒸馏恢复性能。

### 11.2 蒸馏 vs 量化

- 蒸馏改变训练目标
- 量化改变数值表示精度

两者也经常联用。

### 11.3 蒸馏 vs 模仿学习

二者有重叠，但不完全相同。

- 蒸馏更强调从老师模型迁移知识
- 模仿学习更强调学习专家行为

在生成模型和 RL 里，两者经常融合在一起。

### 11.4 数据蒸馏 vs 知识蒸馏

- 数据蒸馏压缩的是数据集
- 知识蒸馏压缩的是模型行为或表示

不要把两者混为同义词。

## 12. 一段简明总结

蒸馏的本质，是把“强模型学到的东西”以一种更容易优化、更容易部署的方式转移给弱模型。

其中：

- logit 蒸馏关注输出分布
- 数据蒸馏关注训练数据的构造或压缩
- on-policy distillation 关注当前策略分布上的在线监督
- self-distillation 关注模型自身内部或代际之间的知识迁移

如果只用一句话概括：

蒸馏不是单一算法，而是一整类“让学生从老师那里更高效地学习”的训练方法。

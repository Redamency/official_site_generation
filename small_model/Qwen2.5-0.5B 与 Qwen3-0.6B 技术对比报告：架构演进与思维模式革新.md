# Qwen2.5-0.5B 与 Qwen3-0.6B 技术对比报告：架构演进与思维模式革新

## 1. 报告综述与核心定位

> Qwen2.5 与 Qwen3 系列代表了从“规模扩展驱动”向“推理范式驱动”的技术跨越。Qwen2.5-0.5B 作为轻量级密集模型，旨在通过极致的预训练数据规模（18T Tokens）提供稳健的通用语言处理与指令遵循能力。相比之下，Qwen3-0.6B（及其代表性的 4B 变体）不仅在数据规模上翻倍至 36T Tokens，更在架构层面引入了原生“思维模式”（Thinking Mode）与 token 级推理协同机制（RelayLLM），实现了从小参数模型向具备主动推理决策能力的智能体演进。

## 2. 模型架构深度对比

Qwen3 在 Qwen2.5 经典的 Transformer 解码器结构基础上，通过解耦嵌入层与精细化的偏差（Bias）控制，显著提升了长文本外推能力与训练稳定性。

| 核心组件 | Qwen2.5 实现方案 | Qwen3 架构革新 |
| :--- | :--- | :--- |
| \*\*嵌入层 (Embedding)\*\* | 采用权重绑定 (Tied Embedding) 以节省参数空间。 | 采用解耦嵌入 (Untied Embedding)，提升语义表达的精确度。 |
| \*\*位置编码 (RoPE)\*\* | 标准旋转位置编码，依赖常规精度计算。 | 采用 \*\*FP32 精度\*\*计算 RoPE 逆频率矩阵，确保位置感知的极高精度。 |
| \*\*偏差处理 (Bias)\*\* | 在 QKV 及 FFN 层广泛使用偏差项。 | 采用 \*\*PaLM 式去偏\*\*设计，但特意在 \*\*QKV 层保留偏差\*\*以优化外推表现。 |
| \*\*稳定性组件\*\* | 依赖标准 RMSNorm 进行预归一化。 | 新增 \*\*QK-Norm\*\* 机制，结合 RMSNorm 极大增强了大模型训练的收敛稳定性。 |
| \*\*激活函数\*\* | SwiGLU，Feed-forward 维度为 $8/3 \cdot H$。 | 延续 SwiGLU，并进一步优化了长文本下的内存管理与计算效率。 |

## 3. 预训练数据规模与多语言策略

Qwen3 系列在数据广度与深度上确立了新的行业基准：

\*   \*\*规模翻倍\*\*：Qwen2.5 预训练于 18 万亿 (18T) Tokens；Qwen3 则跃升至 \*\*36 万亿 (36T)\*\* Tokens，通过更密集的知识洗礼强化了逻辑底座。
\*   \*\*多语言覆盖\*\*：Qwen3 支持 \*\*119 种语言\*\*及方言。其基于 \`cl100k\` 增强的 BPE 分词器，词表规模达到 \*\*152,000\*\*，针对中文及多语言环境进行了特定的压缩效率优化。
\*   \*\*数据混合策略\*\*：
    \*   Qwen2.5 侧重于利用 Qwen2-Instruct 模型作为质量过滤器，进行多维度评分。
    \*   Qwen3 采用了更复杂的混合策略，不仅包含针对代码与数学的上采样，还引入了针对推理链（CoT）的特定合成数据，以配合其新增的思维模式。

## 4. 后训练阶段与强化学习路径

在对齐阶段，Qwen3 从单纯的偏好优化转向了具备验证机制的强化学习。

### Qwen2.5 路径
其后训练流程包含超过 100 万样本的高质量 SFT。强化学习主要采用两阶段：\*\*离线 DPO\*\*（直接偏好优化）用于建立人类偏好基准，\*\*在线 GRPO\*\*（群体相对策略优化）用于在无奖励模型的情况下提升推理一致性。

### Qwen3 路径
引入了 \*\*RLVR（带验证奖励的强化学习）\*\* 框架。通过结合代码沙盒执行反馈与数学答案自动验证，Qwen3 在处理工具调用与代码逻辑时具备更高的容错率。此外，Qwen3 在 RL 阶段重点强化了 \*\*128K 长文本\*\* 的处理能力与 agent 协同逻辑。

## 5. 核心功能创新：思维模式（Thinking Mode）

Qwen3 引入了原生的“思维模式”，这是其区别于传统小参数模型的最关键技术特征。

\*   \*\*模式显式切换\*\*：通过特殊的指令标签实现“快思考”与“慢思考”的切换。
    \`\`\`text
    /think      # 激活链式推理 (CoT)，模型在输出答案前进行内部思考
    /no\_think   # 快速响应模式，适用于直接问答或常识检索
    \`\`\`
\*   \*\*思维预算 (Thinking Budget)\*\*：这是一种 \*\*Token 限制分配机制\*\*。用户可以为内部推理配置 Token 上限。当推理过程达到预算限制时，模型会生成“停止思考”消息并自动转入最终答案生成阶段，从而实现对推理深度与端到端延迟的精细化控制。

## 6. 推理协同与端侧优化（RelayLLM 与量化）

Qwen3 重新定义了小模型（SLM）在异构模型协作中的地位。

\*   \*\*RelayLLM 协同解码\*\*：Qwen3 作为主动控制器，能够通过生成专用命令主动调用大型模型（如 Qwen3-8B）。在六项数学基准测试中，通过仅对 \*\*1.07%\*\* 的关键 Token 进行协同解码（Relay），RelayLLM 将平均准确率从 42.5% 提升至 \*\*49.52%\*\*。与传统的随机路由器相比，这种 token 级协同实现了 \*\*98.2% 的成本缩减\*\*。
\*   \*\*能力内化\*\*：在“Teacher-Free”实验中发现，由于在训练中习得了专家的思维路径，Qwen3 在不调用外部大模型时，其原生推理能力亦优于同尺寸基准模型。
\*   \*\*LieQ 自适应量化\*\*：Qwen3 采用了层级自适应量化技术。在极低的 \*\*2.05-bit\*\* 位宽下，模型仍能保持 \*\*95.9% 的 FP16 基准精度\*\*，显著优于 GPTQ 或 AWQ，为手机等边缘端设备部署提供了极高性价比。

## 7. 基准测试性能评估

由于 Qwen3-0.6B 的详细细分指标未在所有公开数据中完全披露，本报告采用 Qwen3-4B 的性能跨越作为 Qwen3 架构能力的量化依据。

| 测试集 (基准指标) | Qwen2.5-0.5B (Instruct) | Qwen3-4B (Variant) | 性能增益洞察 |
| :--- | :---: | :---: | :--- |
| \*\*MMLU (通用能力)\*\* | 15.0% (Pro) / 47.5% (Base) | \*\*85.00%\*\* | 思维模式对多学科理解有巨大提振。 |
| \*\*GSM8K (数学推理)\*\* | 49.6% | \*\*86.7%\*\* | 验证奖励（RLVR）显著减少计算错误。 |
| \*\*BBH (复杂逻辑)\*\* | 20.3% | \*\*Comparable to 8B+\*\* | 小模型通过 CoT 展现出越级逻辑能力。 |
| \*\*GPQA-Diamond\*\* | 29.8% | \*\*70.20%\*\* | 高难度逻辑从 58.08 跃升至 70.20。 |

\*\*核心性能洞察：\*\*
1.  \*\*逻辑跃迁\*\*：Qwen3 系列在开启思维模式后，其在 GPQA-Diamond 等研究生水平测试上的表现，使其具备了挑战 10B 以上规模模型的能力。
2.  \*\*效率协同\*\*：RelayLLM 证明了只需调用极少数“专家 Token”，即可弥补小模型在 AIME 等竞赛级数学题上的天然短板。
3.  \*\*内化效应\*\*：Qwen3 在后训练阶段习得了更优的推理模式，使其在未开启外部协同的情况下，基准分仍普遍高于 Qwen2.5 对应版本。

## 8. 结论：从规模领先到范式转换

对比 Qwen2.5-0.5B 与 Qwen3-0.6B，技术路径的演进路径清晰：Qwen2.5 标志着\*\*预训练数据规模化扩展\*\*的阶段性巅峰；而 Qwen3 则通过\*\*思维模式与协同解码\*\*实现了范式转移。

\*\*规模驱动能力，思维重塑智能。\*\*
